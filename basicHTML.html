Element Form:

It is used to collect user input. It has two main atributtes:

action
It is the URL where the form data will be sent. Regularly this is a API or a server-site script that will process the data. It is also important and preferable to use absolute paths.
method
It is the HTTP method used to send the data. It can be GET or POST
  • GET: sends the data through (and visible) the URL in a query string.

    Somethin like: "https://example.com/action_page.php?name=Juan&age=25" It clearly is not secure. It can only hold around 2000 characters, and can be use when the data is not sensitive.

  • POST: sends the data through the HTTP body request.

    This is more secure than the GET method, as the data is at first not visible; but it is important to mention that if the data is not encrypted it can be intercepted. The solution is to use secure connections such as HTTPS. Also it can hold more data.

This are tags, associated with forms, but they are not just attributes

label
It is used to describe the input field. It is important for accessibility, as it helps screen readers to understand the form
  • for: element's id. The element which will receive the information
  • form: it references the id of the form where it belongs

    It is redundant to use it here, but in cases like buttons the attribute makes sense

input
It is used to create an input field. It has several types:
  • text: a single line text field

    When this one is use, is also peer with placeholder that is the text inside de box, to explain again, what is expected there

  • password: a single line text field, but the text is hidden
  • radio: a radio button
  • checkbox: a checkbox
  • submit: a submit button
  • file: a file upload field
  • hidden: a hidden field
  • date: a date field
  • number: a number field

It also has the required attribute, that makes the field mandatory Or the minlength and maxlength attributes, that limit the number of characters that can be written in the field.
